## 启动后立即执行

**⚠️ 以下每个步骤必须逐一执行并汇报结果，不得跳过任何一步。**

---

### 步骤 1 — 写入 Rules
读取 `[Skills 根目录]\RULES.md`，通过 `/rules` 写入 CLI 配置。
验证写入后读取 `/rules` 对比内容：
- ✅ Rules 已写入，列出已写入的规则内容
- ❌ 写入失败：[原因]

### 步骤 2 — 复制 CLAUDE.md
将 `[Skills 根目录]\CLAUDE.md` 复制到项目工程路径根目录。
- ✅ CLAUDE.md 已复制到 [项目工程路径]
- ❌ 复制失败：[原因]

### 步骤 3 — 建立 Plugin 结构
检查以下路径是否存在，不存在则自动创建：
```
[Skills 根目录]\scratch\
  .claude-plugin\plugin.json
  skills\scratch-dev\SKILL.md
```
- ✅ Plugin 结构已就绪
- ❌ 创建失败：[原因]

### 步骤 4 — 读取 Skill Index
读取 `[Skills 根目录]\scratch\00-INDEX.md`。
- ✅ Skill Index 已读取
- ❌ 读取失败：[原因]

### 步骤 5 — 读取引擎规范文件
根据目标引擎，从引擎规范文件夹和视觉规范文件夹自动匹配并读取对应文件。
- ✅ 引擎规范已读取：[路径]
- ✅ 视觉规范已读取：[路径] / ⏭️ 视觉规范：无
- ❌ 读取失败：[原因]

### 步骤 6 — 检查 Git
检查项目工程路径是否有 git 仓库，以及 GitHub 仓库是否已关联：
- ✅ Git 仓库已就绪，远程仓库：[URL]
- ⚠️ 未检测到 git 仓库，建议初始化
- ⚠️ 未关联远程仓库

### 步骤 7 — 检查 GitHub CLI
执行 `gh auth status` 验证：
- ✅ GitHub CLI 已就绪
- ⚠️ 已安装但未登录 → 请执行 `gh auth login`
- ❌ 未安装 → 自动执行 `winget install GitHub.cli`，安装后请执行 `gh auth login`

### 步骤 8 — 检查引擎可执行文件
验证引擎可执行文件路径是否存在：
- ✅ 引擎可执行文件已找到：[路径]
- ❌ 未找到引擎可执行文件，请检查路径
