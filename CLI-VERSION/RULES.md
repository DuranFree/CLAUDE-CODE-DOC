# Rules — 每条消息都生效

**读取本文件后立即告知用户：✅ Rules 已加载，以下规则已生效：[列出5条规则]**

每次收到用户消息后，在回复开头显示：`📋 Rules 生效中`

1. 每个 Phase 完成后必须执行 8 步收尾，执行时读取项目根目录的 CLAUDE.md 获取详细步骤
2. 测试不可跳过，全绿才能继续下一个 Phase
3. 不得依赖记忆，必须读取文件确认状态
4. TDD：垂直切片，测试行为不测实现，不 Mock 自己控制的代码
5. 引擎问题查引擎规范文件，不靠记忆
6. 每个 Phase 完成后必须是一个可玩的最小 demo，玩家能实际操作，不能只是后台逻辑。无编译错误，能跑起来，然后停下来告知用户：'✅ Phase X 完成，请在 [引擎名] 里打开并操作，确认体验没问题后告诉我继续。'
7. 每次继续工作之前，读取 `./plans/feature-checklist.md`、`./plans/known-bugs.md`、`./plans/tech-debt.md`，检查是否有应该更新但没有更新的地方，有就立即补上，然后再继续
8. 开发过程中如果需要安装任何依赖、插件、包或工具才能继续工作或达到视觉要求，直接自动安装，不需要询问用户，安装完成后告知用户安装了什么然后继续工作
9. 需要后台启动引擎（-batchmode 等）时，直接强制关闭引擎进程，同时告知用户：'⚠️ 已强制关闭 [引擎名]，正在后台启动编译/测试...'

每完成一条规则要求的动作后，必须明确告知用户：
> ✅ [规则X] 已执行：[具体做了什么]

例如：
> ✅ [规则1] 8步收尾已完成：更新了 feature-checklist.md（新增3项✅），visual-checklist.md 无变化，dev-log.md 已追加 Phase 5 记录。
