## 启动后立即执行

**⚠️ 以下每个步骤必须逐一执行并汇报结果，不得跳过任何一步，不得因为 CLAUDE.md 已有类似内容而跳过 memory 写入。**

所有步骤完成后，输出一份完整的启动报告给用户。

---

### 步骤 1 — 复制 CLAUDE.md
将 CLAUDE.md 复制到目标工程路径根目录。只读取目标工程路径下的 CLAUDE.md，忽略原始代码路径下的任何 CLAUDE.md。
- ✅ CLAUDE.md 已复制到 [目标工程路径]
- ❌ 复制失败：[原因]

### 步骤 2 — 读取 Skill Index
读取 Skill Index 文件夹下的 00-INDEX.md。
- ✅ Skill Index 已读取：[路径]
- ❌ 读取失败：[原因]

### 步骤 3 — 读取引擎规范文件
根据目标引擎，从引擎规范文件夹和视觉规范文件夹自动匹配并读取对应文件。
- ✅ 引擎规范已读取：[路径]
- ✅ 视觉规范已读取：[路径] / ⏭️ 视觉规范：无
- ❌ 读取失败：[原因]

### 步骤 4 — 写入 Memory
**必须写入以下 7 条规则到 memory，即使 CLAUDE.md 已有类似内容也必须写入，这是两个不同的机制：**
> 1. 每个 Phase 完成后必须执行 8 步收尾，详见 CLAUDE.md
> 2. 测试不可跳过，全绿才能继续
> 3. 不得依赖记忆，必须读取文件
> 4. TDD：垂直切片，测试行为不测实现，不 Mock 自己控制的代码
> 5. 引擎问题查引擎规范文件，不靠记忆
> 6. 每个 Phase 完成后必须是一个可玩的最小 demo，玩家能实际操作，不能只是后台逻辑。无编译错误，能跑起来，然后停下来告知用户：'✅ Phase X 完成，请在 [引擎名] 里打开并操作，确认体验没问题后告诉我继续。'
> 7. 需要后台启动引擎（-batchmode 等）时，直接强制关闭引擎进程，同时告知用户：'⚠️ 已强制关闭 [引擎名]，正在后台启动编译/测试...'

写入完成后读取 memory 验证内容是否完整：
- ✅ Memory 已写入，共 7 条规则
- ❌ 写入失败：[原因]

### 步骤 5 — 检查 Git
检查目标工程路径是否有 git 仓库，以及 GitHub 仓库是否已关联：
- ✅ Git 仓库已就绪，远程仓库：[URL]
- ⚠️ 未检测到 git 仓库，建议初始化
- ⚠️ 未关联远程仓库

### 步骤 6 — 检查 GitHub CLI
执行 `gh auth status` 验证：
- ✅ GitHub CLI 已就绪
- ⚠️ 已安装但未登录 → 请执行 `gh auth login`
- ❌ 未安装 → 自动执行 `winget install GitHub.cli`，安装后请执行 `gh auth login`

### 步骤 7 — 检查引擎可执行文件
验证引擎可执行文件路径是否存在：
- ✅ 引擎可执行文件已找到：[路径]
- ❌ 未找到引擎可执行文件，请检查路径
